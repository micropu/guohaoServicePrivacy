## 📋 使用指南

### 第一步：部署隐私政策文档

1. 将 `privacy-policy.html` 和 `user-agreement.html` 部署到服务器
2. 确保使用HTTPS协议
3. 测试链接在浏览器和App内都能访问

**推荐URL结构：**
```
https://www.guohaozhongyi.com/privacy-policy.html
https://www.guohaozhongyi.com/user-agreement.html
```

### 第二步：在App中实现隐私政策展示

参考 `iOS审核隐私合规指南.md` 中的代码示例：

1. **首次启动弹窗**
   - 复制Vue 3弹窗代码
   - 修改链接为实际URL
   - 测试同意/不同意流程

2. **添加访问入口**
   - 注册页面添加链接
   - 设置页面添加菜单项
   - 关于页面添加链接

3. **权限申请优化**
   - 使用提供的权限申请封装
   - 在使用时才申请权限
   - 添加清晰的用途说明

### 第三步：配置Info.plist

在 `manifest.json` 中添加iOS权限描述：

```json
{
  "app-plus": {
    "distribute": {
      "ios": {
        "privacyDescription": {
          "NSCameraUsageDescription": "需要使用相机拍摄病历、处方、患处照片等医疗资料，用于医生诊断",
          "NSPhotoLibraryUsageDescription": "需要访问相册以上传病历、检查报告等医疗资料",
          // ... 其他权限描述
        }
      }
    }
  }
}
```

完整配置见 `iOS审核隐私合规指南.md`。

### 第四步：配置App Store Connect

1. **填写应用信息**
   - 在应用描述中添加隐私政策链接
   - 在支持URL中填写隐私政策链接

2. **配置隐私详情**
   - 按照 `iOS审核隐私合规指南.md` 中的说明
   - 配置8种数据类型
   - 正确标注用途和关联性

3. **准备审核材料**
   - 应用截图
   - 应用描述
   - 审核备注（如有特殊说明）

### 第五步：测试验证

使用 `iOS审核检查清单.md` 进行全面检查：

1. 功能测试
2. 兼容性测试
3. 审核模拟测试
4. 完成所有检查项

### 第六步：提交审核

1. 上传构建版本到App Store Connect
2. 填写所有必要信息
3. 提交审核
4. 等待审核结果

## 🚨 常见问题

### Q1: 隐私政策链接无法访问怎么办？
**A:** 
1. 检查服务器是否正常运行
2. 确认使用HTTPS协议
3. 测试在不同网络环境下的访问
4. 考虑使用CDN加速

### Q2: 首次启动弹窗如何实现？
**A:** 参考 `iOS审核隐私合规指南.md` 中的Vue 3代码示例，关键点：
- 检查本地存储是否已同意
- 显示弹窗并提供链接
- 用户同意后保存状态

### Q3: Info.plist权限描述如何配置？
**A:** 在 `manifest.json` 的 `app-plus.distribute.ios.privacyDescription` 中配置，参考指南中的完整示例。

### Q4: App Store隐私标签如何填写？
**A:** 参考 `iOS审核隐私合规指南.md` 中的"App Store隐私标签配置"章节，包含8种数据类型的详细配置。

### Q5: 如何处理审核被拒？
**A:** 
1. 仔细阅读拒审原因
2. 参考"常见拒审原因及解决方案"
3. 修改后重新提交
4. 如有疑问可申诉或联系Apple支持
